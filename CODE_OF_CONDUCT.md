# Code of conduct

OpsMind is a small project run by people who also have day jobs. The code
of conduct below sets a low, clear bar: be useful, be honest, don't be a
jerk. It borrows from the Contributor Covenant without adopting the full
text because the project doesn't have a formal moderation team.

## What we expect

- **Be useful.** Bug reports include reproduction steps. Feature requests
  describe the problem, not just the solution. Reviews suggest specific
  changes, not general dissatisfaction.
- **Be honest.** If you don't know, say so. If a patch is AI-assisted,
  that's fine — attribute it. If you benchmarked something, show the
  numbers; if you didn't, say "I think" instead of "this is faster".
- **Assume good faith.** Most rude-sounding comments on GitHub are
  someone typing on a phone between meetings. Ask what they meant
  before concluding they're hostile.
- **Treat the factory-floor audience with respect.** OpsMind's users are
  shift leads, QA technicians, and production managers. "Obvious"
  things to you may not be obvious to them; that isn't their fault.

## What we don't tolerate

- **Personal attacks.** On the maintainer, contributors, or anyone
  CC'd in an issue. Disagreement with code is fine; disparagement of
  people isn't.
- **Harassment of any kind.** Including targeted pile-ons, unwelcome
  sexual attention, and doxxing. Zero tolerance, no warning.
- **Commercial spam.** OpsMind is not a place to advertise unrelated
  products. PR comments touting SaaS alternatives will be removed.
- **Deliberate bad-faith reports.** Filing issues to waste maintainer
  time is harassment dressed as contribution.

## Scope

This code of conduct applies to:

- issues, PRs, and discussions on <https://github.com/Pawansingh3889/OpsMind>
- comments on code reviews
- public communication about OpsMind on other platforms (conference
  talks, blog posts, social media) where you represent OpsMind

It does **not** apply to private messages between individuals, unrelated
projects, or personal life. If a private interaction is brought up as
context, the maintainer will weigh it case by case.

## Reporting

If someone is being abusive in public, report it by:

1. **Preferred:** open a private security advisory at
   <https://github.com/Pawansingh3889/OpsMind/security/advisories/new>
   and mark it "Code of conduct". The form isn't just for vulnerabilities;
   it's the only private channel we have.
2. **Alternative:** email the maintainer via the address listed on
   <https://github.com/Pawansingh3889> with subject line
   "OpsMind code of conduct".

Include links to the comments in question and a brief description of the
problem. We will acknowledge receipt within **7 days**.

## Enforcement

The maintainer handles enforcement. Escalation path:

1. **Warning** — a private note explaining what was unacceptable and
   what to do differently. Most situations end here.
2. **Temporary block** — for repeated warnings or a single serious
   incident, the offender loses GitHub interaction on the repo for a
   cooling-off period (days to weeks).
3. **Permanent block** — for harassment, doxxing, threats, or abuse
   that resumes after a temporary block. No appeal path.

The maintainer is not a lawyer. Rulings are judgement calls, made in
public comments where appropriate and in private where the target
prefers privacy.

## Changes

This document changes via PR from the maintainer, announced in the PR
description. Community input is welcome in the corresponding issue.
